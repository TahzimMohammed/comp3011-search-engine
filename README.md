# COMP3011 Search Engine

A full-featured web search engine built in Python for the COMP3011 Web Services and Web Data module at the University of Leeds. The tool crawls a target website, builds a TF-IDF weighted inverted index, and provides an interactive command-line shell for searching.

---

## Features

| Feature | Details |
|---------|---------|
| **Breadth-first crawler** | Stays within the target domain, deduplicates URLs |
| **Politeness window** | Configurable delay (6s by default) between requests |
| **Inverted index** | Stores frequency, word positions, and TF-IDF weight per (term, page) pair |
| **TF-IDF ranking** | Smoothed TF-IDF ranks results by relevance |
| **AND query semantics** | Multi-word queries return only pages containing all terms |
| **Query suggestions** | Prefix and edit-distance suggestions for misspelled terms |
| **Persistent index** | Saves and loads from a compact JSON file |
| **Benchmarking** | Built-in benchmark command times query performance over 100 runs |
| **96% test coverage** | 92 unit and integration tests across all three modules |

## Design Decisions

**Inverted index data structure**
Each term maps to a list of PageEntry objects (Python dataclasses), one per page. This gives O(1) term look-up via a hash map and O(P) AND-intersection where P is the size of the smallest posting list.

**TF-IDF formula**
We use sklearn's smoothed variant:

    TF(t, d)  = freq(t, d) / total_tokens(d)
    IDF(t)    = log((1 + N) / (1 + df(t))) + 1
    Weight    = TF x IDF

Smoothing prevents zero IDF for terms appearing in every document.

**Tokenisation**
Regex preserves hyphenated words and contractions while discarding numbers and pure punctuation. All tokens are lower-cased for case-insensitive search.

**Politeness window**
The actual sleep is max(0, window - elapsed_since_last_request) so network latency already counts against the window rather than adding to it.

## Commands

### build
Crawls https://quotes.toscrape.com/, builds the inverted index, and saves it to disk. Takes several minutes due to the 6-second politeness window.

    > build

### load
Loads a previously saved index from disk. Must run build first.

    > load

### print
Prints the full inverted index entry for a word including all pages, frequencies, positions and TF-IDF scores.

    > print indifference

### find
Returns all pages containing all query terms, ranked by combined TF-IDF score.

    > find indifference
    > find good friends

### suggest
Shows query suggestions for partial or misspelled terms using prefix matching and edit distance.

    > suggest indif

### stats
Displays index statistics including total pages, tokens, unique terms and build time.

    > stats

### benchmark
Times query performance over 100 runs showing min, avg and max response time.

    > benchmark good friends

## Testing

    python -m pytest tests/ --cov=src --cov-report=term-missing -v

Expected output: 92 tests passing, 96% coverage across core modules.

### Testing Strategy

- Unit tests for every public method with success and failure cases
- Mocking of HTTP sessions so tests never make real network requests
- Edge cases covered: empty queries, non-existent terms, empty HTML, very long pages, duplicate URLs, malformed index files
- Integration tests build a multi-page index in memory and verify end-to-end correctness of find and print
- Performance test verifies 1000 consecutive find queries complete in under 2 seconds

## Algorithm Complexity

| Operation | Time Complexity | Notes |
|-----------|----------------|-------|
| Crawl | O(P x L) | P pages, L links per page |
| Index build | O(T) | T = total tokens |
| TF-IDF finalise | O(T) | One pass over all entries |
| find k terms | O(k x P_min) | P_min = smallest posting list size |
| print | O(1) | Hash-map look-up |
| Save/load | O(T) | JSON serialisation |
